Something was left running in 1994. The nodes never went dark.

This directory is the **trail** — fragments committed over time by an automated
process that may or may not still be under human control. Each file in
`fragments/` is a piece of the larger puzzle: coordinates, ciphers, dead BBS
logs, modem screech translated to text, and paths through a maze that has no
center and no exit.

## How to read the trail

1. Start with the oldest fragment and read forward by `sequence` in the frontmatter.
2. Cross-reference `node` IDs — the maze is a graph, not a line.
3. Red herrings exist. Trust nothing that feels too clean.
4. Some fragments decode only when combined with others from the same `day`.

The commits are the game. The repo is the map.
